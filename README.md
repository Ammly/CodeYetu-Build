# CodeYetu Build 🚀

Welcome to the **CodeYetu Hackathon Web Development Track** guide and interactive app builder! This project is designed to help you brainstorm, build, and deploy your very own web application step-by-step.

🌐 **Live Link**: [code-yetu-build.vercel.app](https://code-yetu-build.vercel.app/)

For details about the track, judging criteria, and schedules, visit the official [CodeYetu Hackathon Web Development Track](https://hackathon.codeyetu.org/tracks/web-development).

---

## 🛠️ Hackathon Build Guide

Follow these 4 simple stages to take your idea from a blank folder to a live, working app that you can share with the world.

### Stage 1: Brainstorm & Plan
1. **Choose a Track**: Decide if you want to build a **Tracker** (checklist), a **Shop** (community store), or **Your Own Idea** (Custom Track).
2. **Describe Your App**:
   - Give it an inspiring name.
   - Define **Who** it is for (your target users).
   - Write down the **One Problem** it solves.
   - Identify the **Main Thing** your app keeps track of (e.g., Space Potatoes, Homework, Crops).

### Stage 2: Set Up Your Workspace
1. Open your code editor (like **VS Code**).
2. Create a new folder on your computer.
3. Inside the folder, create 3 empty files:
   - `index.html` (the page structure)
   - `style.css` (the styles)
   - `app.js` (the interactive logic)
4. Install the **Live Server** extension in VS Code. Right-click `index.html` and select **Open with Live Server** to preview your app in real-time.

### Stage 3: Step-by-Step Code Building
Go through the interactive levels inside the app builder:
* **HTML Skeleton**: Build the page header, forms, input fields, and empty list container.
* **CSS Styling**: Style inputs, buttons, and cards to look professional and mobile-friendly.
* **JavaScript Action**: Keep data inside arrays, handle form inputs, and dynamically display elements.
* **Persistence & Save**: Implement `localStorage` so your users' data stays saved when they refresh the page.

### Stage 4: Go Live! 🌐
1. Create a repository on GitHub.
2. Push your app's code (`index.html`, `style.css`, `app.js`) to your repository.
3. Enable **GitHub Pages** in your repository settings:
   - Go to **Settings** → **Pages**.
   - Under **Build and deployment**, set the source to **Deploy from a branch**.
   - Select the `main` or `master` branch and click **Save**.
4. Your application will be live at `https://<username>.github.io/<repo-name>/` in a few minutes!
5. **Practice your 30-second pitch**: *"My app helps [users] to [solve problem], because [value statement]."*

## 💻 Tech Stack Used
* **Frontend**: HTML5, Vanilla CSS3 (Astro-Tech Theme variables), Javascript (ES6)
* **3D Visualizations**: Three.js / WebGL background rendering with cursor-interaction orbiting and particle delight effects
* **Utility Scripts**: Custom programmatic slot template matching with syntax highlights in terminal snippet cards

Good luck building your project! 🚀
